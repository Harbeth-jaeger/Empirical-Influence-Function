# Go Single-Line FIM 数据处理说明

本文档用于向学长交付当前 Go single-line FIM 实验的数据口径、文件位置、处理脚本与推荐命令。当前主线只讨论 Go 语言；多语言扩展后续再补。

## 0. 数据格式和命名约定

数据格式约定：

- `*_canonical.jsonl`：保留 `prefix / target / suffix / full_code / metadata / judge_payload` 等结构化字段，适合评测、oracle、自检和后续分析。
- `*_chatml.jsonl`：把同一条样本渲染成项目统一 ChatML-MASK 推理/训练格式，模型输入为包含 `[MASK]` 的代码，模型输出只预测缺失代码。
- `*_compact.jsonl` ：在 ChatML 样本上额外加入 token、label、attention_edges 等结构化因果/依赖标注，用于 saliency alignment 或图信号训练。

当前三套主数据状态：


| 数据                     | canonical | chatml | compact | 备注                                                        |
| ------------------------ | --------: | -----: | ------: | ----------------------------------------------------------- |
| train 10K                |     10000 |  10000 |   10050 | compact 已合并为单文件；由前 5K 左右标注 + 后 5K 标注组成。 |
| CodeSearchNet valid full |      7611 |   7611 |    7611 | 全量标注完成。                                              |
| CodeSearchNet test full  |      7979 |   7979 |    7979 | 全量标注完成。                                              |

## 1. Train Data：训练数据

### 1.1 当前使用的数据文件

当前 Go single-line FIM 训练主文件：


| 文件                                                                            | 样本数 | 用途                                                        |
| ------------------------------------------------------------------------------- | -----: | ----------------------------------------------------------- |
| `data/go_single_fim/train_data/go_single_train_v2_canonical.jsonl`              |  10000 | 训练数据的结构化 canonical 版本，包含 gold`target`。        |
| `data/go_single_fim/train_data/go_single_train_v2_chatml.jsonl`                 |  10000 | 训练使用的 ChatML-MASK 版本。                               |
| `data/go_single_fim/train_data/go_single_train_v2_graphsignal_10k_compact.json` |  10050 | 训练数据的 compact 标注版本，带`attention_edges` 等标注边。 |

训练样本来自 CodeSearchNet Go。处理逻辑是从 Go 函数中筛选无注释、非测试、非生成代码的函数，再抽取单行 statement 作为 `[MASK]` 的 `target`。目标 statement 类型主要包括 assignment、return、call。

### 1.2 标注状态

10K train 的标注已经整理为最终单文件 compact。最终 compact 来自“前 5K 左右标注样本”与“后 5K 标注样本”的合并；后 5K 原始标注过程中失败 3 条，但最终保留文件统一命名为 10K。


| 标注文件                                                                        |  行数 | 状态                                                            |
| ------------------------------------------------------------------------------- | ----: | --------------------------------------------------------------- |
| `data/go_single_fim/train_data/go_single_train_v2_graphsignal_10k_compact.json` | 10050 | train 10K 最终 compact 标注文件，带`attention_edges` 等标注边。 |

当前 `train_data` 最终只保留 3 个文件：canonical、chatml、compact。

## 2. Internal Eval：同分布 CodeSearchNet Go 评测数据

### 2.1 当前使用的数据文件

Internal eval 使用 CodeSearchNet Go 的 valid/test split，和训练数据同源、同任务形态，都是 single-line FIM completion。valid full、test full 的 canonical/chatml/compact 都已经准备好。


| split | canonical 文件                                                             | chatml 文件                                                             | compact 标注文件                                                                    | 样本数 |
| ----- | -------------------------------------------------------------------------- | ----------------------------------------------------------------------- | ----------------------------------------------------------------------------------- | -----: |
| valid | `data/go_single_fim/eval_data/codesearchnet_go_valid_full_canonical.jsonl` | `data/go_single_fim/eval_data/codesearchnet_go_valid_full_chatml.jsonl` | `data/go_single_fim/eval_data/codesearchnet_go_valid_full_srcannotate_compact.json` |   7611 |
| test  | `data/go_single_fim/eval_data/codesearchnet_go_test_full_canonical.jsonl`  | `data/go_single_fim/eval_data/codesearchnet_go_test_full_chatml.jsonl`  | `data/go_single_fim/eval_data/codesearchnet_go_test_full_srcannotate_compact.json`  |   7979 |

### 2.2 是否带标注

canonical/chatml 本身不带因果标注，只带 completion gold target。compact 文件带 token、label、attention_edges 等标注，可用于 saliency alignment。当前 internal eval 标注状态：


| 文件                                                                                | 行数 | 状态                  |
| ----------------------------------------------------------------------------------- | ---: | --------------------- |
| `data/go_single_fim/eval_data/codesearchnet_go_valid_full_srcannotate_compact.json` | 7611 | valid full 标注完成。 |
| `data/go_single_fim/eval_data/codesearchnet_go_test_full_srcannotate_compact.json`  | 7979 | test full 标注完成。  |

### 2.3 测什么指标

因为 CodeSearchNet internal split 没有官方 unit tests，所以 internal eval 不报告 unit-test pass。当前 evaluator 的 completion 指标是：

- `pass@1`：模型第 1 个 completion 与 gold target 做 normalized exact match。
- `pass@10`：前 10 个 completion 中是否有任意一个与 gold target normalized exact match。
- `CodeBLEU`：把 `prefix + prediction + suffix` 和 `prefix + target + suffix` 拼回 full code 后计算。

如果输入了带 `attention_edges` 的 compact 标注文件，还可以评测 saliency alignment：

- `recall@k`
- `precision@k`
- `mAP@k`

### 2.4 相关代码文件


| 脚本                                                                  | 作用                                                                                                                                        |
| --------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| `scripts/data_process/build_codesearchnet_go_full_eval.py`            | 从 CodeSearchNet Go raw valid/test 构造 full internal eval canonical/chatml。每个函数只选一个 mask target，避免一个函数膨胀成多条评测样本。 |
| `scripts/go_singleline_fim_exp/evaluate_codesearchnet_go_internal.py` | internal eval 入口；支持 oracle、prediction JSONL、CodeBLEU、以及基于 annotated compact 的 saliency alignment。                             |
| `scripts/go_singleline_fim_exp/annotate_chatml_with_src_annotate.py`  | 给 internal valid/test ChatML 样本加结构化标注。                                                                                            |

### 2.5 推荐命令

重新构造 CodeSearchNet Go valid/test full eval：

```bash
python scripts/data_process/build_codesearchnet_go_full_eval.py \
  --raw-root data/go_single/raw_data/codesearchnet/go/final/jsonl \
  --output-dir data/go_single_fim/eval_data \
  --report-dir outputs/go_singleline_fim_exp/reports \
  --splits valid,test
```

oracle 检查 completion 指标是否为 1：

```bash
python scripts/go_singleline_fim_exp/evaluate_codesearchnet_go_internal.py \
  --oracle \
  --eval-data data/go_single_fim/eval_data/codesearchnet_go_test_full_chatml.jsonl \
  --annotated-data data/go_single_fim/eval_data/codesearchnet_go_test_full_chatml.jsonl \
  --skip-saliency \
  --compute-codebleu \
  --run-name codesearchnet_go_test_full_oracle \
  --output outputs/go_singleline_fim_exp/eval_results/codesearchnet_go_test_full_oracle_results.jsonl \
  --summary outputs/go_singleline_fim_exp/eval_results/codesearchnet_go_test_full_oracle_summary.json \
  --table-md outputs/go_singleline_fim_exp/eval_results/codesearchnet_go_test_full_oracle_table.md
```

使用模型预测文件评测 completion：

```bash
python scripts/go_singleline_fim_exp/evaluate_codesearchnet_go_internal.py \
  --predictions outputs/<your_predictions>.jsonl \
  --eval-data data/go_single_fim/eval_data/codesearchnet_go_test_full_chatml.jsonl \
  --annotated-data data/go_single_fim/eval_data/codesearchnet_go_test_full_chatml.jsonl \
  --skip-saliency \
  --compute-codebleu \
  --pass-k 10
```

如需跑 saliency alignment，把 `--annotated-data` 换成带 `attention_edges` 的 compact 标注文件，并提供模型路径：

```bash
python scripts/go_singleline_fim_exp/evaluate_codesearchnet_go_internal.py \
  --predictions outputs/<your_predictions>.jsonl \
  --eval-data data/go_single_fim/eval_data/codesearchnet_go_test_full_chatml.jsonl \
  --annotated-data data/go_single_fim/eval_data/codesearchnet_go_test_full_srcannotate_compact.json \
  --model_name_or_path outputs/<your_model_or_adapter> \
  --compute-codebleu \
  --pass-k 10 \
  --top-k 10
```

## 3. External Benchmark：外部 benchmark 测试数据

### 3.1 当前使用的数据文件

当前外部主评测暂时只使用两个 Go benchmark derived FIM：


| benchmark                  | canonical 文件                                                                    | chatml 文件                                                                    | 样本数 | 是否带标注                                 |
| -------------------------- | --------------------------------------------------------------------------------- | ------------------------------------------------------------------------------ | -----: | ------------------------------------------ |
| HumanEval-X Go derived FIM | `data/go_single_fim/test_data/humaneval_x/humaneval_x_go_derived_canonical.jsonl` | `data/go_single_fim/test_data/humaneval_x/humaneval_x_go_derived_chatml.jsonl` |    129 | 不带因果标注；带 unit test judge payload。 |
| MCEval Go single FIM       | `data/go_single_fim/test_data/mceval/mceval_go_single_v2_canonical.jsonl`         | `data/go_single_fim/test_data/mceval/mceval_go_single_v2_chatml.jsonl`         |     45 | 不带因果标注；带 unit test judge payload。 |

二者合计 174 条外部测试样本。

### 3.2 测什么指标

External benchmark 有 judge payload，可以运行 Go unit tests，因此主指标是：

- `pass@1`：第 1 个 completion 拼回完整代码后是否通过单测。
- `pass@10`：前 10 个 completion 中是否有任意一个通过单测。
- `CodeBLEU`：greedy completion 拼回 full code 后与 oracle full code 对比。

当前 oracle 自检结果已经通过：


| scope          | 样本数 | pass@1 | pass@10 | CodeBLEU |
| -------------- | -----: | -----: | ------: | -------: |
| HumanEval-X Go |    129 |    1.0 |     1.0 |      1.0 |
| MCEval Go      |     45 |    1.0 |     1.0 |      1.0 |
| overall        |    174 |    1.0 |     1.0 |      1.0 |

oracle 输出文件：

- `outputs/go_singleline_fim_exp/eval_results/oracle/go_external_fim_oracle_results.jsonl`
- `outputs/go_singleline_fim_exp/eval_results/oracle/go_external_fim_oracle_summary.json`

### 3.3 为什么暂时没有 MultiPL-E

MultiPL-E 是权威的多语言 code generation benchmark，但当前公开 HuggingFace parquet 中 Go 数据主要提供 `prompt / tests / stop_tokens`，没有 Go `canonical_solution` 字段。

我们的当前任务不是原生“根据 prompt 生成完整函数体”，而是 derived 的 Go single-line FIM：需要从 reference solution 里诚实地抽取一行 statement 作为 gold `target`，再构造 `prefix / target / suffix`。没有 Go canonical solution 时，无法构造严格的 single-statement gold target，也无法计算 completion-level CodeBLEU reference。

因此当前不把 MultiPL-E 放入主评测。之前生成过的 body-mask 临时文件只作为探索产物，不作为当前论文主结果：

- `data/go_single_fim/test_data/multipl_e/multipl_e_go_bodymask_canonical.jsonl`
- `data/go_single_fim/test_data/multipl_e/multipl_e_go_bodymask_chatml.jsonl`
