## 三、核心脚本

### 1. 一键封装脚本

```text
scripts/huawei_deploy/annotate.sh
```

支持模式：

```bash
bash scripts/huawei_deploy/annotate.sh prepare     # raw prompt/response/task_id -> ChatML/canonical
bash scripts/huawei_deploy/annotate.sh validate    # prepare 后检查 ChatML/FIM 格式
bash scripts/huawei_deploy/annotate.sh check       # prepare -> validate -> 标注 CHECK_ROWS 条 -> 默认生成可视化
bash scripts/huawei_deploy/annotate.sh visualize   # 对 check 结果生成 CSV/HTML
bash scripts/huawei_deploy/annotate.sh full        # prepare -> validate -> 全量标注
bash scripts/huawei_deploy/annotate.sh all         # check，可选可视化，然后 full
```

`prepare` 不会覆盖原始数据；转换后的 ChatML/canonical 会写到 `data/huawei_data/processed_*`。如果目标 ChatML/canonical 已存在，脚本默认跳过转换；需要强制重建时才设置：

```bash
export FORCE_PREPARE=1
```

### 2. 队友辅助脚本

```text
scripts/go_singleline_fim_exp/check_chatml_fim_input.py
scripts/go_singleline_fim_exp/run_huawei_annotation.sh
```

`check_chatml_fim_input.py` 检查每行是否有 `uid / language / target / messages`，并检查 user message 是否含 `[MASK]`、assistant message 是否等于 `target`。

`run_huawei_annotation.sh` 是底层标注入口，只支持 `check/full`。外层 `scripts/huawei_deploy/annotate.sh` 会自动调用它，现场优先用外层脚本。

## 四、华为 API 环境变量

明天先打开队友留下的：

```bash
cat EIF-huawei-annotation/常用命令.md
```

把里面真实值填到下面占位符。

### 0. 选择 API 模式

在华为绿区机器上走华为网关，使用默认模式或显式设置：

```bash
export REQUIRE_HUAWEI_GATEWAY=1
```

本地只测试普通 OpenAI-compatible 云端模型、没有华为网关信息时，才设置：

```bash
export REQUIRE_HUAWEI_GATEWAY=0
```

`REQUIRE_HUAWEI_GATEWAY=0` 时，脚本不会要求 `HW_APPKEY/HW_OPERATOR`，也不会发送 `X-HW-ID/X-HW-APPKEY/appId/scene/operator` 等华为网关字段。明天到华为侧正式标注时应使用 `REQUIRE_HUAWEI_GATEWAY=1`。

### 1. OpenAI-compatible 基础变量

```bash
export OPENAI_BASE_URL="https://apigw-cn-southe2.huawei.com/api/v1"
export OPENAI_API_KEY="<api_key_from_常用命令.md>"
export ANNOTATE_MODEL="<deployed_model_name_from_常用命令.md>"
```

含义：

- `OPENAI_BASE_URL`：华为 OpenAI-compatible 网关地址。
- `OPENAI_API_KEY`：OpenAI-compatible client 使用的 key；即使华为网关还要求额外 header，这个变量也要有。
- `ANNOTATE_MODEL`：实际部署的标注模型名。

### 2. 华为网关身份变量

```bash
export HW_ID="com.huawei.ipd.coretool.coreai"
export HW_APPKEY="<x_hw_appkey_from_常用命令.md>"
export HW_APP_ID="com.huawei.ipd.coretool.coreai"
export HW_SCENE="test"
export HW_OPERATOR="<operator_or_work_id_from_常用命令.md>"
```

代码会把这些字段传给华为接口：

- 请求 header：`X-HW-ID`、`X-HW-APPKEY`。
- 请求 body：`appId`、`scene`、`operator`。

### 3. HTTP、解码和重试变量

```bash
export ANNOTATE_HTTP_PROXY_NONE=1
export ANNOTATE_VERIFY_SSL=0
export HW_ENABLE_THINKING=0
export ANNOTATE_TEMPERATURE=0.2
export ANNOTATE_MIN_REQUEST_INTERVAL=1.0
export ANNOTATE_MAX_RETRIES=8
export ANNOTATE_RETRY_BASE_SLEEP=10
export ANNOTATE_MAX_TOKENS=2048
```

含义：

- `ANNOTATE_HTTP_PROXY_NONE=1`：不走代理，适配绿区网络。
- `ANNOTATE_VERIFY_SSL=0`：关闭 SSL 校验，适配华为网关证书环境。
- `HW_ENABLE_THINKING=0`：传入 `chat_template_kwargs.enable_thinking=false`。
- `ANNOTATE_TEMPERATURE=0.2`：标注模型采样温度。
- `ANNOTATE_MIN_REQUEST_INTERVAL`：请求间隔；如果 429，可以调大到 `2.0`。
- `ANNOTATE_MAX_RETRIES / ANNOTATE_RETRY_BASE_SLEEP`：失败重试策略。
- `ANNOTATE_MAX_TOKENS`：单次标注请求最大输出 token。

如果华为侧还要求额外 header/body，使用：

```bash
export ANNOTATE_EXTRA_HEADERS_JSON='{"Header-Name":"value"}'
export ANNOTATE_EXTRA_BODY_JSON='{"customField":"value"}'
```

## 九、重要参数解释

- `HUAWEI_RAW_DATA`：华为原始 jsonl，字段是 `prompt / response / task_id`。
- `HUAWEI_CHATML_DATA`：转换后的 ChatML/FIM jsonl，给标注脚本使用。
- `HUAWEI_CANONICAL_DATA`：转换后的 canonical jsonl，保留原始 id、prefix、target、suffix、full code 等追溯信息。
- `OUT_DIR`：compact 标注数据输出目录。按当前约定，华为 compact 放在 `data/huawei_data/processed_*`。
- `RUN_DIR`：日志和 annotation cache 输出目录。
- `VIS_OUT_DIR`：CSV/HTML 可视化输出目录，默认 `outputs/huawei_deploy`。
- `CHECK_ROWS`：check 模式标注多少条，明天建议 30。
- `STRIP_CJK_COMMENTS=1`：转换时删除 Go 代码里包含中文字符的注释，保留非中文注释。
- `MAX_TARGET_NONEMPTY_LINES / MAX_TARGET_ROUGH_TOKENS / MAX_TARGET_CHARS`：转换时过滤过长 completion；当前推荐 `10 / 192 / 1024`，约保留 75.8% 样本。
- `NUM_WORKERS`：并发请求数。先试 12，不行降 8，再不行降 4。
- `ANNOTATION_MODE=agent`：推荐标注模式；API 不支持 tool calling 时，代码会 fallback 到普通 JSON 对话补边。
- `MODEL_PATH`：本地 Qwen tokenizer 路径，用来把边映射到 Qwen BPE token。主要用 tokenizer，不一定加载完整模型权重。
- `OPENAI_BASE_URL / OPENAI_API_KEY / ANNOTATE_MODEL`：OpenAI-compatible API 调用三件套。
- `REQUIRE_HUAWEI_GATEWAY`：是否走华为网关；华为侧正式部署设 `1`，本地普通云端模型测试设 `0`。
- `HW_ID / HW_APPKEY / HW_APP_ID / HW_SCENE / HW_OPERATOR`：华为网关鉴权、审计和业务路由字段；仅 `REQUIRE_HUAWEI_GATEWAY=1` 时需要。
- `ANNOTATE_EXTRA_HEADERS_JSON / ANNOTATE_EXTRA_BODY_JSON`：华为侧临时新增字段的兜底入口。

## 十、排查提示

### 1. 缺少 `HW_APPKEY` 或 `HW_OPERATOR`

当 `REQUIRE_HUAWEI_GATEWAY=1` 时，日志会在标注前退出，例如：

```text
Set HW_APPKEY to the Huawei X-HW-APPKEY value.
```

说明还没进入真正标注。去 `EIF-huawei-annotation/常用命令.md` 复制真实值后重跑。如果只是在本地用普通 OpenAI-compatible 云端模型测试，设置 `REQUIRE_HUAWEI_GATEWAY=0`。

### 2. 429 / rate limit

优先降低并发：

```bash
export NUM_WORKERS=8
```

还不稳就：

```bash
export NUM_WORKERS=4
export ANNOTATE_MIN_REQUEST_INTERVAL=2.0
```

### 3. 输入格式不对

先跑：

```bash
bash scripts/huawei_deploy/annotate.sh validate
```

要求每行有 `uid / language / target / messages`，user message 必须含 `[MASK]`，assistant message 必须等于 `target`。

### 4. 模型 tokenizer 路径不存在

检查：

```bash
ls -lh "$MODEL_PATH"
```

如果华为侧路径不同，重新设置：

```bash
export MODEL_PATH=/path/to/Qwen2.5-Coder-7B-Instruct
```

## 十一、华为 completion 长度分析

本地新增了只读分析脚本，用来统计华为 raw 数据中 completion 的长度分布和中文注释比例：

```bash
python scripts/huawei_deploy/analyze_huawei_completion_lengths.py \
  --input-path data/huawei_data/cloud_core_test_25.JunJunly_GoOnly_length_filter.jsonl \
  --report-output outputs/huawei_deploy/huawei_completion_length_report.json \
  --long-sample-output outputs/huawei_deploy/huawei_long_completion_samples.jsonl
```

当前本地统计结果：

- 总样本：199923。
- `completion` 非空行数 p50=5，p75=10，p90=18，p95=24。
- 如果用 `>3` 行过滤，会过滤约 65.3%，过于激进。
- 推荐中等阈值：`lines<=10 && rough_tokens<=192 && chars<=1024`，约保留 75.8%。
- 只过滤极端长样本可用：`lines<=20 && rough_tokens<=256 && chars<=2048`，约保留 92.4%。
